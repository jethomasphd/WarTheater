## I. MISSION BRIEF

Build the definitive public intelligence dashboard for the 2026 Iran War — a live-updating, AI-powered web application that tracks the financial and humanitarian consequences of the conflict in real time. This is not a news aggregator. This is not a blog with charts. This is a **situation room for the American public** — the civilian equivalent of what gets beamed into the White House basement.

**The model**: The Johns Hopkins COVID-19 Dashboard. That tool became the single authoritative reference point for a global crisis. It didn't editorialize. It didn't persuade. It presented the numbers — infections, deaths, recoveries — and let the weight of the data do the talking. 270 million people used it. We build that, but for war.

**The thesis**: Every bomb has a price tag. Every missile has a human cost. Every day the Strait of Hormuz stays closed, a gas station in Georgetown, Texas charges more per gallon. This dashboard makes those connections visible, visceral, and undeniable.

---

## II. WHAT THIS IS (AND IS NOT)

**This IS:**
- A real-time conflict intelligence dashboard with live data feeds
- A financial impact tracker connecting military operations to economic consequences
- A humanitarian cost counter — casualties, displacement, infrastructure destruction
- A geographic visualization of the theater of operations
- A public utility — open, free, fast, mobile-first
- AI-enhanced with daily briefings generated by Claude via Cloudflare Worker proxy

**This is NOT:**
- An editorial platform or opinion site
- A pro-war or anti-war advocacy tool
- A news aggregator or RSS reader
- A social media experience
- A static HTML page with hardcoded data

### Core Components

1. **Frontend** — Single-page application deployed to Cloudflare Pages
   - Framework: Vanilla JS + Leaflet.js + Chart.js + D3.js (no React — keep it lean and fast)
   - CSS: Custom design system, no framework
   - Target: <500KB initial load, <2s First Contentful Paint

2. **Cloudflare Worker** — API proxy + AI briefing engine
   - Proxies requests to Claude API (Anthropic) for daily intelligence briefings
   - Proxies requests to external data APIs (ACLED, oil prices, etc.)
   - Handles CORS, rate limiting, API key management
   - Caches responses in Cloudflare KV with appropriate TTLs

3. **Cloudflare KV** — Persistent key-value store
   - Caches API responses (15-minute TTL for prices, 1-hour for conflict data, 24-hour for AI briefings)
   - Stores historical snapshots for trend visualization
   - Stores the daily AI briefing so it's generated once, served many times

4. **External Data Sources** (fetched server-side through Worker):
   - **ACLED API** — Conflict events, fatalities, actor data for Iran/Iraq/Israel/Lebanon/Gulf states
   - **Oil Price API** — Real-time Brent and WTI crude prices (oilpriceapi.com or commodities-api.com)
   - **FRED API** — Federal Reserve Economic Data (gas prices, CPI, treasury yields)
   - **FlightAware / Flightradar24** — Flight cancellation data (or scrape from public sources)
   - **UNHCR Refugee Data API** — Displacement statistics
   - **HDX Humanitarian API** — Cross-referenced humanitarian indicators
   - **Wikipedia / Wikidata** — Structured conflict timeline data (for timeline enrichment)
   - **News RSS feeds** — Major wire services for latest headline context

### Domain & Deployment

- **Domain**: `wartheater.org` or `thewartheater.com` (register through Cloudflare)
- **Hosting**: Cloudflare Pages (free tier supports this)
- **Worker**: Cloudflare Workers (free tier: 100K requests/day — sufficient for launch)
- **KV**: Cloudflare KV (free tier: 100K reads/day, 1K writes/day)
- **SSL**: Automatic via Cloudflare
- **CDN**: Automatic via Cloudflare's global edge network

## IV. THE FIVE PANELS

The dashboard is organized into five primary intelligence panels, each accessible via persistent top-level navigation. The default view is Panel 1 (Theater Map) with Panel 2 (Financial Impact) visible below the fold.

### Panel 1: THE THEATER (Interactive Map)

The existing war theater map is the foundation. It gets promoted from static HTML artifact to the live centerpiece.

**Geography**: Iran, Persian Gulf, Levant, Arabian Peninsula, Eastern Mediterranean
**Base map**: Dark CartoDB tiles (desaturated to near-black) with Stamen terrain overlay at low opacity for topographic context
**Projection**: Mercator (standard for Leaflet)

**Layer System** (toggleable):
- **Strike Sites — US/Israel → Iran** (ember orange pulsing dots)
  - Each dot: city name, known targets, reported casualties, date of first strike
  - Special marker: Minab girls' school (white, civilian casualty site)
  - Data: Manually curated initial dataset + ACLED API enrichment
- **Retaliation Sites — Iran → Gulf/Region** (deep red pulsing rings)
  - Each dot: base name, host country, what was hit, casualties
  - Data: Manually curated + ACLED
- **Missile Arcs** (animated parabolic trajectories)
  - Red arcs: Iranian outbound (toward Gulf targets + Israel)
  - Blue/white arcs: US/Israeli inbound (toward Iran)
  - Animated at slow sweep speed — luminous, not frantic
- **Carrier Strike Groups** (constellation clusters)
  - USS Abraham Lincoln, USS Gerald R. Ford, USS George H.W. Bush
  - Approximate positions updated via open-source naval tracking
- **Strait of Hormuz** (amber chokepoint glow)
  - Pulsing closure indicator
  - Hover: "20% of global oil transits this waterway"
- **Hezbollah / Lebanon Front** (violet markers)
  - Beirut strikes, Radwan Force targets
  - Faint violet streaks for attack waves toward northern Israel
- **Missile Range Rings** (faint concentric circles from Tehran)
  - Shahab-3: ~2,000km
  - Fattah: ~1,400km
  - Kheibar Shekan: ~1,450km
- **Humanitarian Layer** (toggle on/off)
  - Refugee flow arrows (from UNHCR data)
  - Displacement camps
  - Hospital/infrastructure damage sites

**Timeline Scrubber**: Bottom of map. Day 1 (Feb 28) → Today. Tappable day dots (not a slider). Scrubbing progressively lights up strike sites. Key event labels on major days.

**Key Event Labels**:
- Feb 28: "Operation begins — Khamenei killed"
- Feb 28: "Iran retaliates on Gulf states"
- Mar 1: "Strait of Hormuz closure"
- Mar 2: "Hezbollah enters war"
- Mar 2: "6 US service members KIA in Kuwait"
- Mar 7: "Oil strikes begin"
- Mar 9: "Mojtaba Khamenei named Supreme Leader"
- [Auto-populate future events from ACLED + news feeds]

### Panel 2: THE COST (Financial Impact)

This is where the dashboard differentiates from every other conflict tracker. **Every military action has an economic shadow**, and this panel makes it visible.

**Hero Metrics** (large, real-time, updating):
- **Brent Crude Price** — live, with % change since Feb 27 (pre-war baseline)
- **WTI Crude Price** — live, with % change
- **US Gas Price (national avg)** — from AAA/FRED, with % change
- **S&P 500** — with % change since Feb 27
- **US Defense Spending (running estimate)** — extrapolated from DoD operational cost data
- **Estimated Global Economic Cost** — aggregate model (oil price shock × GDP sensitivity)

**Charts**:
- **Oil Price Timeline** — Brent + WTI from Jan 1 2026 to present, with conflict start marked as vertical line. Pre-war trend line extrapolated forward to show divergence.
- **Gas Price Heatmap** — US state-level gas prices (if data available) or regional breakdown
- **Market Impact** — S&P 500, defense stocks (RTX, LMT, NOC, GD, BA), oil majors (XOM, CVX, COP), airlines (DAL, UAL, AAL) — indexed to Feb 27 = 100
- **Hormuz Chokepoint** — Visualization of daily tanker transit volume (historical vs. current near-zero)
- **Cost Per Day** — Estimated daily US military operational cost (modeled from historical analogues: Iraq War ~$720M/day at peak)
- **Consumer Impact Calculator** — Interactive: "How much more are YOU paying?" Input: miles driven per week, home heating type, state → output: estimated additional monthly cost

### Panel 3: THE TOLL (Humanitarian Impact)

The human cost. No euphemisms. No abstraction. Numbers with names where possible.

**Hero Counters** (animated count-up on load):
- **Total Strikes** (US/Israel on Iran): 3,000+ and counting
- **Iranian Casualties** (reported): updating from ACLED + news sources
- **US Service Members KIA**: 6 (and updating)
- **US Service Members Wounded**: 10+ (and updating)
- **Lebanese Casualties**: 486+ (and updating)
- **Displaced Persons**: 450,000+ in Lebanon, plus Iranian internal displacement
- **Flights Cancelled**: 14,000+ (and updating)
- **Civilian Infrastructure Destroyed**: hospitals, schools, bridges, power plants

**Visualizations**:
- **Casualty Timeline** — Stacked area chart by day, disaggregated by category (Iranian military, Iranian civilian, US military, Lebanese, other)
- **Displacement Flow** — Sankey diagram or flow arrows showing population movement FROM conflict zones TO neighboring countries
- **Infrastructure Damage** — Icon grid showing types of civilian infrastructure confirmed destroyed (hospital, school, power plant, water treatment, bridge, etc.)
- **Historical Context** — Small multiple comparing Day N of this conflict to Day N of: Iraq War (2003), Libya (2011), Afghanistan (2001), Gulf War (1991)

### Panel 4: THE BRIEFING (AI-Powered Daily Intelligence)

This is the COMPANION Protocol in action. A daily intelligence briefing generated by Claude, delivered through the Cloudflare Worker proxy.

**Daily Briefing** (generated once per day, cached in KV):
- **Situation Summary**: 3-paragraph plain-language overview of the last 24 hours
- **Key Developments**: Bullet list of 5-7 most significant events
- **Financial Outlook**: How markets reacted, what to watch
- **Humanitarian Update**: Latest displacement, casualty, and infrastructure data
- **What to Watch**: Forward-looking analysis of next 24-48 hours
- **Sources**: Links to primary sources used

**Implementation**:
```javascript
// Worker: /api/briefing
// 1. Check KV for today's briefing
// 2. If not found or stale (>12 hours):
//    a. Fetch latest ACLED data
//    b. Fetch latest oil prices
//    c. Fetch latest news headlines (RSS)
//    d. Construct prompt with all data
//    e. Call Claude API (claude-sonnet-4-20250514)
//    f. Cache response in KV with 12-hour TTL
// 3. Return cached briefing
```

**Prompt Template** (stored in Worker):
```
You are an intelligence analyst preparing a daily briefing on the 2026 Iran War for public consumption. You are precise, factual, and unflinching. You do not editorialize. You present the data and its implications.

Today is [DATE]. Here is the latest data:

CONFLICT DATA (from ACLED):
[ACLED_DATA]

FINANCIAL DATA:
- Brent crude: $[PRICE] ([CHANGE]% since Feb 27)
- WTI crude: $[PRICE] ([CHANGE]% since Feb 27)
- US gas avg: $[PRICE]/gal ([CHANGE]% since Feb 27)
- S&P 500: [VALUE] ([CHANGE]% since Feb 27)

LATEST HEADLINES:
[RSS_HEADLINES]

Generate a structured intelligence briefing with these sections:
1. SITUATION SUMMARY (3 paragraphs, plain language)
2. KEY DEVELOPMENTS (5-7 bullet points)
3. FINANCIAL OUTLOOK (2 paragraphs)
4. HUMANITARIAN UPDATE (2 paragraphs)
5. WHAT TO WATCH (3-5 forward-looking items)

Be direct. Be precise. Let the data speak.
```

### Panel 5: THE RECORD (Timeline + Archive)

A comprehensive, scrollable timeline of every significant event since Feb 28. This becomes the historical record.

**Format**: Vertical timeline, left-aligned, with event cards
**Each card**:
- Date/time
- Event title
- 2-3 sentence description
- Category tag (military, financial, humanitarian, diplomatic, domestic)
- Source link
- Associated data point (e.g., "Oil +4.2% this day")

**Data**: Combination of:
- Manually curated initial timeline (from existing war theater data)
- ACLED event feed (auto-populated)
- AI-enriched summaries (from daily briefings)

**Filter controls**: By category, by date range, by geography

### Visual Treatments

- **Scanline overlay**: Full-viewport `::before` pseudo-element with repeating gradient
- **Vignette**: Radial gradient from transparent center to dark edges
- **Corner brackets**: Military-style frame markers in corners of major panels
- **Glow effects**: `box-shadow` with color-matched translucent layers, not CSS filters
- **Panel borders**: 1px solid with subtle inner glow on hover/active states
- **Classification banner**: Top of page — "UNCLASSIFIED // PUBLIC INTELLIGENCE DASHBOARD"
- **Live indicator**: Blinking green dot next to "LIVE" label on real-time data
- **Cursor coordinate tracker**: Desktop only — shows lat/lng as cursor moves over map

### Responsive Strategy

- **Desktop (>1024px)**: Full theater view. Map takes 65% width, financial panel takes 35% right column. Panels stack below.
- **Tablet (768-1024px)**: Map full-width. Panels in scrollable tab interface below.
- **Mobile (<768px)**: Map full-viewport with bottom sheet pattern. Swipe between panels. Tap-to-inspect markers. Timeline becomes horizontal scroll of day dots.

### Accessibility

- Color is never the only differentiator (shapes + labels accompany all color coding)
- All charts have `aria-label` descriptions
- Keyboard navigation for all interactive elements
- Minimum 4.5:1 contrast ratio for body text (WCAG AA)
- Screen reader announcements for live-updating data

## VI. CLOUDFLARE WORKER — DETAILED SPECIFICATION

### File: `worker/src/index.js`

```
Routes:
  GET  /api/oil          → Proxy to oil price API, cache 15 min
  GET  /api/conflict     → Proxy to ACLED API, cache 1 hour
  GET  /api/displacement → Proxy to UNHCR API, cache 6 hours
  GET  /api/markets      → Proxy to financial data, cache 15 min
  GET  /api/briefing     → Generate or serve cached AI briefing
  GET  /api/timeline     → Serve curated + auto-populated timeline
  GET  /api/gas-prices   → Proxy to FRED/AAA data, cache 1 hour
  POST /api/ask          → (Future) User questions answered by Claude
```

### Worker Environment Variables (Secrets)

```
ANTHROPIC_API_KEY=sk-ant-...
ACLED_API_KEY=...
OIL_API_KEY=...
FRED_API_KEY=...
ALLOWED_ORIGINS=https://wartheater.org,https://www.wartheater.org,http://localhost:8788
```

### KV Namespace

```
Binding: THEATER_CACHE
Keys:
  oil:latest          → {brent, wti, timestamp}
  oil:history         → [{date, brent, wti}, ...]
  conflict:latest     → {events: [...], timestamp}
  displacement:latest → {figures: {...}, timestamp}
  briefing:YYYY-MM-DD → {html, generated_at}
  timeline:events     → [{date, title, desc, category, source}, ...]
  gas:national        → {price, change, timestamp}
  markets:snapshot    → {sp500, defense_stocks, oil_stocks, airlines, timestamp}
```

### Rate Limiting Strategy

- Oil prices: Fetch every 15 minutes max (free tier limits)
- ACLED: Fetch every hour (weekly update cadence from source)
- AI briefing: Generate once per day (or twice: morning + evening)
- All other: Serve from KV cache, refresh on TTL expiry
- Client-side: Poll `/api/oil` every 5 minutes, others every 30 minutes
